Workflow

Add References from a collection's References tab

This mockup is for end-user validation before development. It focuses on the M42 scope: direct expression entry, optional source or collection seed, version and reference type selection, Bulk Add for concept or mapping IDs, include/exclude, cascade, preview, and submit feedback.

1 Open from HEAD

User starts on a collection's References tab and clicks Add References.

2 Seed or type

User can select a source or collection to create a base path, or type an expression directly.

3 Choose target

Version and Reference Type determine whether IDs expand as concepts or mappings.

4 Preview

Preview evaluates expressions and surfaces version mismatch, zero-result, and duplicate context.

5 Add

The dialog submits expressions, then shows success and failure rows inline.

In scope for this mockup

Expression mode, Bulk Add mode, Concepts and Mappings, Include and Exclude, cascade selection, locked-version warning, preview, and submit completion.

Out of scope for this mockup

Structured intensional filter builder, search-and-select concept browsing, and the final enhanced preview API behavior. Those remain design or API dependencies.

Validation states

Specific cases to walk through with users

Direct expression

User does not select a source. They type or paste a full expression and add it as a single reference.

Concept Bulk Add

User selects CIEL, keeps Reference Type as Concepts, pastes IDs, previews duplicates and missing IDs, then submits.

Mapping Bulk Add

User switches Reference Type to Mappings and verifies that ID expansion targets /mappings/{id}/, not /concepts/{id}/.

Version mismatch

User changes from Locked to Latest or a specific version and sees a warning before preview and submit.

Exclude reference

User chooses Exclude and confirms that the dialog communicates include: false without changing the rest of the workflow.

Released version

User views a released collection version and sees why Add References is unavailable until switching to HEAD.

Review questions

What we need to learn before development

Workflow clarity

Can users predict what will be submitted when both Expression and Bulk Add have values?

Mapping support

Do mapping authors understand that Bulk Add accepts mapping IDs after selecting Reference Type = Mappings?

Preview trust

Does the preview give enough confidence without implying that all final API behavior is already implemented?

Mock data is illustrative. Final implementation should use the enhanced references preview endpoint once it supports batch expressions, new/existing grouping, version status, and typed errors.
