Workflow

Add References from a collection's References tab

This mockup is for end-user validation before development. It focuses on the M42 scope: Add by ID(s), optional direct expression entry, source or collection seed, version and reference type selection, include/exclude, cascade, preview, and submit feedback.

1 Open from HEAD

User starts on a collection's References tab and clicks Add References.

2 Seed or type

User can select a source or collection to create a base path, or type an expression directly.

3 Choose target

Version and Reference Type determine whether IDs expand as concepts or mappings.

4 Preview

Preview evaluates expressions and surfaces version mismatch, zero-result, and duplicate context.

5 Add

The dialog submits expressions, then shows success and failure rows inline.

In scope for this mockup

Add by ID(s), Expression mode, Concepts and Mappings, Include and Exclude, versionless resolution, CascadeSelector-style cascade/transform controls, manual preview, and submit completion.

Out of scope for this mockup

Structured intensional filter builder, embedded concept/mapping browsing, released-version editing, and the final enhanced preview API behavior. Those remain design or API dependencies.

Validation states

Specific cases to walk through with users

Direct expression

User does not select a source. They type or paste a full expression and add it as a single reference.

Concept Add by ID(s)

User selects CIEL, keeps Reference Type as Concepts, pastes IDs, previews duplicates and missing IDs, then submits.

Mapping Add by ID(s)

User switches Reference Type to Mappings and verifies that ID expansion targets /mappings/{id}/, not /concepts/{id}/.

Version mismatch

User changes from Versionless to a specific pinned version that differs from the collection's versionless resolution and sees a warning before preview and submit.

Exclude reference

User chooses Exclude and confirms that the dialog communicates include: false without changing the rest of the workflow.

Browse selected repo

User opens the selected source or collection when they need to search for concepts or mappings before adding.

Review questions

What we need to learn before development

Workflow clarity

Can users tell whether they are in Add by ID(s) mode or Expression mode, and why only one can be active?

Mapping support

Do mapping authors understand that Add by ID(s) accepts mapping IDs after selecting Reference Type = Mappings?

Preview trust

Does manual preview plus refresh make it clear when results are current after the user changes fields?

Mock data is illustrative. Final implementation should use the enhanced references preview endpoint once it supports batch expressions, new/existing grouping, version status, and typed errors.
